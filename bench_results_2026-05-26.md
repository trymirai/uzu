# NF4 LUT QMV Bench — 5-run Aggregated Results

**Source:** 5 bench runs from `/tmp/simdbar_devbuf_run*.log`
**Hardware:** Apple M4 (GPU family 9)
**Config:** BITS=4, GROUP_SIZE=64, bf16 activations, batched timing (N=128 dispatches/CB, 5 warmup + 20 measured, drop 5 farthest from median)

All Δ values are percentage points vs the scalar AWQ baseline (`QmvFast use_lut=false use_zero_points=true`). Cells show `Δ% ± run-σ_pp` aggregated across 5 runs.

Key variants:
- **scalar** — AWQ baseline (no LUT, int4 mantissa bit-trick)
- **awq-lut256** — AWQ with closed-form bypass (production-winner for AWQ)
- **tmpl-awq** — AWQ with template specialization (slightly faster than awq-lut256)
- **nf4-tg-sbar-dev** — NF4 with per-simdgroup LUT slicing + device-buffer codebook (THE NF4 WIN, ship-worthy)
- **nf4-tg-sbar** — Same per-simdgroup pattern, codebook from constant array (same perf as devbuf within ±0.5pp)
- **nf4-tg** — Prior best NF4 (16-entry shared TG codebook)
